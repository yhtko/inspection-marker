# Swimlane Flowchart

部署・担当者ごとのスイムレーンを持つフローチャートを、ブラウザ上で作成するWebツールです。

サーバー送信は行わず、すべてブラウザ内で処理します。

## 主な機能

- スイムレーン追加、削除、並べ替え
- レーン名編集
- レーン背景色のプリセット選択
- レーン内クリックで工程ノード追加
- ノード形状選択
  - 長方形
  - 角丸長方形
  - ひし形
- ノードのドラッグ移動
- ノードのダブルクリック編集
- ノード間の矢印接続
- 接続線ラベルのダブルクリック編集
- Deleteキーまたは右クリックで削除
- JSON保存 / JSONロード
- PNG出力
- SVG出力

## セットアップ

```powershell
cd C:\conda\PDFDrawing
npm install
```

## 開発実行

```powershell
npm run dev
```

表示されたURLをブラウザで開きます。

## 本番ビルド

```powershell
npm run build
```

ビルド結果は `dist` フォルダに出力されます。

## 操作

- 左サイドバーの `レーン追加` でレーンを追加
- レーン名欄を編集して名称変更
- `上` / `下` でレーン順を変更
- 色パネルでレーン背景色を変更
- 図形を選択して、レーン内をクリックするとノードを追加
- ノードをドラッグして移動
- ノードをダブルクリックしてテキスト編集
- ノード右側のハンドルから別ノードへドラッグして接続
- 接続線をダブルクリックして `Yes` / `No` などのラベルを編集
- ノードまたは接続線を選択して `Delete` キーで削除
- 右クリックメニューでも削除可能

## 出力

- `JSON保存`: レーン、ノード、接続線、表示位置を保存
- `JSONロード`: 保存したJSONを読み込み、編集再開
- `PNG出力`: 現在表示しているキャンバスをPNG保存
- `SVG出力`: 図の内容をSVG保存

## GitHub Pagesへ公開する場合

このリポジトリにはGitHub Actionsによる自動公開設定を含めています。

1. GitHubでリポジトリを作成します。
2. このプロジェクトのソース一式をpushします。
3. GitHubの `Settings` → `Pages` を開きます。
4. `Build and deployment` の `Source` を `GitHub Actions` にします。
5. `main` ブランチへpushすると、自動で `npm ci` と `npm run build` が実行され、`dist` がPagesへ公開されます。

## 制限事項

- デスクトップブラウザ向けです。
- PDF編集機能はありません。
- draw.ioのような多機能作図ツールではなく、スイムレーンフロー作成に絞っています。
- PNG出力は現在の表示範囲を保存します。
- SVG出力は内部データから簡易生成します。ReactFlowの曲線表示と完全一致しない場合があります。
